# 프로젝트 코드 작성 지침

## 주석 작성 원칙

- 새 소스 파일을 만들거나 기존 파일을 크게 변경할 때는 파일 상단이나 핵심 클래스 근처에 이 파일이 맡는 역할을 한두 줄로 요약한다.
- 공개 클래스, 공개 메서드, API, 그리고 동작을 바로 이해하기 어려운 내부 헬퍼에는 사람이 나중에 읽고 검토할 수 있도록 무엇을 하는 코드인지와 어디에서 어떻게 쓰이는지 짧게 설명한다.
- 검토자가 동작을 검증하는 데 도움이 된다면 중요한 입력값, 반환값, 부수 효과, 제약 조건, 기대 사용 방식을 함께 적는다.
- 주석은 구현을 그대로 반복하기보다 의도와 사용 맥락을 설명하는 데 집중한다. 이름과 코드 구조만으로 충분히 드러나는 내용은 굳이 다시 쓰지 않는다.
- Java, Kotlin, Python 등 언어별 표준 문서화 방식이 있다면 해당 형식(Javadoc, KDoc, docstring 등)을 우선 사용한다.
