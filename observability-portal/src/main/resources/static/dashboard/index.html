Observation Portal

Starter-first observability dashboard

Spring Boot 앱에 starter를 붙이면 project, application, instance 단위로 수집 연결과 운영 상태를 확인합니다. 이 도구는 starter-first observability dashboard입니다.

Project Entry는 운영 판단 화면이 아니라 scope 선택 화면입니다. Application Dashboard가 primary first-screen이며 Project Entry와 Application List는 좁혀 들어가는 navigation surface입니다.

GitHub OAuth only entry입니다. callback은 service access token을 dashboard memory로만 전달하고 resource API는 Authorization: Bearer header로 호출합니다. 이 화면은 브라우저 저장소나 URL token 전달을 요구하지 않습니다.

1

Starter 연결

dependency, portal base URL, project key placeholder, environment 설정을 먼저 이해합니다.

2

Project 등록

로그인 후 Project Entry에서 project를 등록하고 starter project key를 생성 시 1회만 확인합니다.

3

Dashboard 확인

서버가 준 Application List와 Dashboard link를 따라갑니다. accepted bucket freshness와 starter heartbeat는 별도 source로 표시됩니다.

Onboarding

Project 등록

GitHub OAuth로 로그인한 account 안에서 project를 만들고, starter가 `X-OBS-Project-Key`로 사용할 credential을 생성 시 1회만 표시합니다.

credential은 생성/회전 성공 직후에만 표시됩니다. 화면을 떠나면 다시 볼 수 없고 필요하면 새로 회전해야 합니다.

Project card에서 credential metadata, rotation, revocation을 선택할 수 있습니다.

Setup Guide

Starter 연결

  1. 1

    Dependency

    com.sst:observability-spring-boot-starter:0.1.0-SNAPSHOT
  2. 2

    Portal base URL

    observation.heartbeat.portal-base-url=https://portal.example.internal
  3. 3

    Starter project key

    observation.heartbeat.project-key=<X-OBS-Project-Key-placeholder>

    Portal resource API의 Authorization: Bearer token과 분리된 starter ingest credential입니다.

  4. 4

    Environment

    observation.metric-flush.environment=local
