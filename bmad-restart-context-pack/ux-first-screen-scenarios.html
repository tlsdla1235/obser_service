Direct Ingest UX Example

State-first triage desk를 예시 화면으로 먼저 잠근다

이 HTML은 architecture 이전에 first-screen semantics를 잠그기 위한 시각 예시다. 핵심은 차트보다 먼저 상태를 읽고, insight가 없더라도 이유와 next step을 분명히 보여주는 것이다.

이 페이지가 고정하는 것

  • 상태와 freshness가 항상 first read라는 점
  • `0~3 insight`와 `0 insight`의 다른 의미들
  • `Split Desk`가 next-check surface라는 점
  • `Calm Ops Desk` 톤과 가벼운 recovery-first 흐름
1. Waiting First Data 2. Insufficient Sample 3. Nothing Worth Surfacing 4. Active Triage 5. Stale Recovery

Scenario 5

Stale Recovery

이상 판단보다 recovery 안내가 먼저 오는 상태. stale와 down을 같은 감정으로 보여주지 않는다.

RECOVERY FIRST
checkout-api prod ingest paused
last ingest: 2m 11s ago
STALE
최근 데이터가 늦게 들어오고 있습니다.

서비스 상태를 단정하기엔 이릅니다. 먼저 ingest 경로와 앱 실행 상태를 확인해야 합니다.

Next step: 앱 프로세스, network egress, project key 설정 순서로 확인하세요.

Recovery checklist

1. 앱 프로세스가 살아 있는지 확인합니다.

2. outbound HTTPS 경로가 막히지 않았는지 확인합니다.

3. 최근 배포로 설정이 바뀌지 않았는지 확인합니다.
