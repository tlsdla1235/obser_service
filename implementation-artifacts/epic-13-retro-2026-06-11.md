# Epic 13 Retrospective - Dashboard Source of Truth Realignment

## Epic Review

Epic 13의 목적은 확정된 Dashboard Source of Truth를 Application Dashboard, Snapshot, Instance Dashboard, Retention 흐름에 안전하게 적용하는 것이었다. 이 epic은 완료된 Epic 4/5/6/10 story를 다시 여는 작업이 아니라, 새 alignment story에서 `Aligns`, `Supersedes`, `Hardens` 관계로 추적하는 정렬 epic이었다.

회고 기준은 `implementation-artifacts/sprint-status.yaml`, `_bmad/custom/project-context.md`, `planning-artifacts/epics.md`, `planning-artifacts/dashboard-source-of-truth-realignment-roadmap.md`, `planning-artifacts/dashboard-source-of-truth-realignment-sequence.yaml`, Story 13.2부터 13.11의 Dev Agent Record와 Completion Notes다. Epic 12 회고 파일은 별도로 존재하지 않아 이전 회고 follow-through는 수행하지 않는다.

현재 closure 판단은 가능하다. `sprint-status.yaml` 기준 Epic 13, Story 13.1부터 13.11, documentation tracking story 13-doc-0부터 13-doc-3가 모두 `done`이며, 남은 항목은 본 회고였던 `epic-13-retrospective`뿐이었다.

## 잘 된 점

| 주제 | 관찰 | 반복 가치 |
|---|---|---|
| Guard-first 진행 | 13.2 frontend guard가 P2 이후 backend/frontend 변경을 fail-closed로 감시했다. | 큰 UI/API 정렬 전에 source/order/state guard를 먼저 두는 방식은 효과적이었다. |
| Phase와 rollback 단위 분리 | P1부터 P10까지 frontend, backend, snapshot, instance, retention을 작은 vertical slice로 나눴다. | cross-cutting realignment도 phase별로 닫으면 완료 story를 되돌리지 않고 추적할 수 있다. |
| Source boundary 언어 통일 | `accepted_metric_buckets`, `dashboard_snapshots.read_model_json`, `current_window_end_utc`, `recent_30_minutes`가 story, tests, UI copy에 반복됐다. | source 이름을 화면 copy와 test assertion에 함께 남기면 drift를 빨리 발견할 수 있다. |
| Snapshot/current fallback 차단 | snapshot detail, instance snapshot mode, retention expired path에서 live/current fallback을 만들지 않는 guard가 반복됐다. | "없으면 현재값으로 보완" 유혹을 막는 negative AC가 계속 필요하다. |
| Instance surface 분리 | live evidence, selected snapshot evidence reconstruction, stored trend projection을 서로 다른 surface로 분리했다. | instance 화면이 application state owner를 침범하지 않도록 하는 좋은 패턴이다. |
| Review finding의 guard화 | browser/demo evidence 과장, endpoint evidence 과노출, marker/order/retention guard 같은 finding이 completion notes와 regression guard로 남았다. | review 결과는 단발 문구 보정이 아니라 다음 실행을 막는 guard로 환원하는 편이 안전하다. |

## Next Epic Preparation

Epic 14가 아직 명확히 정의되어 있지 않더라도, Epic 13에서 넘겨야 할 준비 기준은 분명하다.

- 새 dashboard/UI story는 `guard:read-model-contract`를 먼저 통과시키고, UI가 state/order/source를 재계산하지 않는지 확인한다.
- Snapshot 또는 retention 관련 후속 작업은 `current_window_end_utc`와 `bucket_end_utc + 30분 evidence grace` 의미를 바꾸지 않는다.
- Instance 관련 후속 작업은 live, selected snapshot reconstruction, stored trend projection을 같은 surface로 합치지 않는다.
- Cleanup rollout은 기본 disabled/dry-run 의미를 유지하고, production enablement는 별도 runbook 또는 decision record로 닫는다.
- Browser demo 보강은 Source of Truth 재정의나 완료 story rollback이 아니라 fixture/runbook/evidence gap closure로 다룬다.
