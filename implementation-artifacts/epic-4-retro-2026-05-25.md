# Epic 4 Retrospective - State Semantics and Time Windows

## 1. 회고 범위

이 회고는 Epic 4 종료 판단과 Epic 5/6 read model 및 dashboard flow 준비를 위해 작성한다. 근거 산출물은 `sprint-status.yaml`, Epic 3 회고, Epic 4 context, Story 4.0-4.4, bridge story 4.0.1/4.0.2, 그리고 최신 contract 문서다.

Epic 4의 active 범위는 아래 네 가지다.

- starter heartbeat와 metric bucket ingest의 source-of-truth 분리
- UTC 30초 bucket, query 시점 current/baseline 15분 window, accepted bucket freshness helper
- `LifecycleStateService`의 accepted bucket metric state와 starter connection axis 분리
- recovery guidance와 state semantics regression guard

Epic 4 범위가 아닌 항목은 dashboard read model/API, UI, dashboard snapshot/history, operational event history, endpoint priority, full insight rule engine, app/project/window p95/p99 rollup이다. 이 항목은 Epic 5/6 준비 항목으로만 다룬다.

### 중요한 closure 해석

Epic 4 closure는 "dashboard가 완성됐다"는 뜻이 아니다. Epic 4는 후속 dashboard/read model이 믿고 소비할 상태 언어와 시간 의미를 닫은 epic이다.

닫힌 기준은 아래와 같다.

- accepted bucket은 metric freshness, metric state, read model의 data-plane source-of-truth다.
- starter heartbeat는 starter/application process liveness, portal reachability, project key validity, metadata validity의 별도 control-plane source다.
- heartbeat 성공/실패/미수신은 accepted bucket freshness, host business health, dashboard snapshot/event, p95/p99, rule/read-model calculation source가 아니다.
- 최근 heartbeat와 없음/오래된 accepted bucket 조합을 host application down으로 단정하지 않는다.
- recovery copy와 starter connection copy를 결합해 host application down 같은 확정 표현을 만들지 않는다.
- `summary.localPercentiles`는 source-scoped starter canonical percentile point이며 여러 instance/window p95/p99를 평균/병합하지 않는다.
- UI, controller, repository는 lifecycle state, insight rule, p95/p99, endpoint priority를 재계산하지 않는다.

## 9. Next Epic Preparation

Epic 5는 Epic 4가 닫은 state semantics를 소비해 server-computed read model/API를 만든다. 준비 판단은 "state language is trustworthy"에서 "read model response is the only UI truth"로 초점이 바뀐다.

준비 체크:

- `DashboardReadModelService` 후보는 `LifecycleStateService` 결과를 재판정하지 않고 조립한다.
- accepted bucket freshness는 마지막 accepted bucket `endUtc` 기준을 유지한다.
- heartbeat telemetry는 starter connection/liveness input으로만 들어간다.
- `sourceScopedPercentiles`는 `accepted_metric_buckets.local_percentiles_json`의 starter canonical point를 노출한다.
- `triageCards=[]`와 recovery/idle/telemetry unreachable mapping은 Epic 5 Story 5.4에서 닫는다.
- snapshot/history는 stored dashboard read model 기반 bounded history로 구현하고 raw explorer처럼 보이게 만들지 않는다.

Significant discovery: Epic 5/6 방향을 중단시킬 방향 전환은 없다. 다만 Epic 4가 남긴 핵심 발견은 "dashboard가 계산을 더 많이 하는 것"이 아니라 "server read model이 이미 계산한 의미를 정확히 표시하는 것"이 Epic 5/6의 성공 조건이라는 점이다.
