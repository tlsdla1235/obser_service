Observation Context 및 Low Cardinality Guard 설명

이 문서는 프로젝트를 처음 보는 리뷰어가 Micrometer HTTP observation 입력이 어떤 경계를 거쳐 안전한 endpoint rollup과 ingest payload로 바뀌는지 빠르게 이해하도록 정리한 PR 설명용 요약이다.

한 줄 요약

Micrometer HTTP observation을 method + normalizedRoute 중심의 low-cardinality 모델로 닫고, raw path, query string, high-cardinality tag가 rollup 및 ingest payload로 넘어가지 않도록 guard를 적용했다.

Observation Context에서 읽는 값

starter는 Micrometer Observation.Context 전체를 그대로 신뢰하지 않고, HTTP 서버 요청 관측에 필요한 low-cardinality 후보만 꺼낸다.

사용하는 low-cardinality 값

method, status, http.route, uri/path, exception

사용하지 않는 값

http.url, high-cardinality tag, custom context map, userId, tenantId, sessionId, traceId

http.route는 framework가 제공한 route template 후보로 최우선 사용한다. uri/path는 http.route가 없거나 blank/UNKNOWN일 때만 allowlist matching을 위한 임시 raw path candidate가 된다.

Low Cardinality Normalize 대상

정규화의 핵심 대상은 endpoint rollup key를 구성하는 method와 normalizedRoute다. endpoint key는 method + normalizedRoute만으로 만든다.

  • method는 허용된 HTTP method 또는 UNKNOWN으로 제한한다.
  • normalizedRoute는 framework route template, allowlist template, UNKNOWN 중 하나다.
  • raw path, query key/value, 사용자 식별자, tenant/session/trace 값은 endpoint key에 들어가지 않는다.

Guard 흐름

guard는 observation 입력을 rollup 가능한 내부 모델로 바꾸는 안전 경계다. 이 경계를 지나면 downstream 코드는 raw path나 arbitrary tag map을 받을 수 없다.

1 Micrometer binder

Observation.Context에서 필요한 후보 값만 추출한다.

2 Input boundary

HttpServerObservationInput에 query 없는 route/path 후보만 둔다.

3 Low-cardinality guard

RouteNormalizationService로 route를 정규화하고 실패하면 UNKNOWN으로 수렴한다.

4 Rollup model

LowCardinalityHttpServerObservation에는 normalized route만 남는다.

중요: http.route가 present지만 invalid shape인 경우에는 uri/path allowlist fallback으로 우회하지 않고 UNKNOWN 처리한다.

Route Allowlist와 Query String 정책

allowlist는 query string allowlist가 아니라 route template allowlist다. 설정 namespace는 observation.route-attribution.allowlist이며, 아래처럼 template만 선언한다.

observation: route-attribution: allowlist: - /orders/{orderId} - /search

허용 예시

  • /orders/{orderId}
  • /api/v1/orders/{orderId}
  • /search

거부 예시

  • /orders/12345처럼 실제 ID가 들어간 raw path
  • /orders/{orderId}?debug=true처럼 query string이 포함된 template
  • https://example.test/orders/{orderId} 같은 absolute URL

raw path candidate의 query string은 matcher 입력 전에 버린다. 이는 query를 정규화하거나 허용한다는 뜻이 아니라, query key/value를 어떤 route, tag, metric key, payload, log, read model에도 남기지 않는다는 뜻이다.

PR에 바로 붙일 짧은 문구

Micrometer HTTP observation을 method + normalizedRoute 중심의 low-cardinality 모델로 닫고, raw path/query/high-cardinality tag가 rollup 및 ingest payload로 넘어가지 않도록 guard를 추가했다. Route allowlist는 query string 허용 목록이 아니라 http.route 부재 시 raw path를 안전한 route template으로 귀속시키기 위한 template 목록이며, query string은 matching 전에 폐기된다.
